## 测试步骤

### 测试 1：初始页面加载
**预期**：
1. 看到 "PHILLY CULTURAL RADAR" 标题
2. 看到 "SEARCH FOR VENUES" 大标题
3. 看到搜索输入框、"Libraries" 和 "Museums" 按钮
4. 输入框自动获得焦点

**如果失败**：

### 测试 2：搜索场馆（快速搜索）
**操作**：点击 "Libraries" 按钮

**预期**：
1. 按钮立即显示 "⊙ SEARCHING..."
2. 按钮被禁用（灰色）

4. 页面切换到搜索结果视图

- 响应内容是否包含有效的 JSON？

## 常见问题排查

### 问题：搜索返回空结果
```
症状：点击 "Libraries" 后，看到 "No results found"
原因：API 返回了错误响应或无效 JSON
解决：
1. 检查 API Key 是否正确
2. 检查浏览器控制台是否有 "Parse error"
3. 尝试在 console 中手动测试：
   > searchVenues("library Philadelphia")
```

### 问题：爬取卡住（SCRAPING… 不动）
```
症状：看到脉动的 "SCRAPING…" 但一直不完成
原因：API 调用超时或被阻止
解决：
1. 刷新页面
2. 检查网络连接
3. 检查 OpenRouter API 状态
4. 查看浏览器控制台的错误信息
```

### 问题：显示 "FAILED" 红色错误
```
症状：某些场馆显示红色的 ✗ FAILED
原因：该场馆的爬取 API 调用失败（正常）
解决：
- 这是可以接受的，其他场馆继续爬取
- 该场馆不会显示在最终的数据表格中
```

### 问题：表格为空
```
症状：跳转到 Layer 3 但没有事件数据
原因：所有场馆都失败，或 API 返回了空事件数组
解决：
1. 回到 Layer 2，检查是否有 "COMPLETE" 状态的场馆
2. 尝试不同的搜索词（如 "library Boston"）
3. 检查浏览器控制台的爬取错误日志
```

## 浏览器开发者工具检查清单

### Console（控制台）
- [ ] 搜索时是否看到 "Search error" 或 "Parse error"？
- [ ] 爬取时是否看到 "Scrape error"？
- [ ] 是否有红色的 JS 错误？

### Network（网络）
- [ ] 是否有 POST 请求到 `openrouter.ai`？
- [ ] 响应状态是否为 200？
- [ ] 响应体是否包含有效的 JSON？

### Performance（性能）
- [ ] 搜索和爬取的响应时间是否合理（1-5 秒）？
- [ ] 脉动动画是否流畅？
